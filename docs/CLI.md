# CLI EFD Unpacker

Приложение поддерживает два сценария запуска из командной строки.

## 1. GUI-режим

Передайте `.efd` файл без команды `unpack`, и приложение откроет GUI с уже выбранным файлом.

```bash
efd_unpacker /path/to/file.efd
```

На Windows вместо условного имени `efd_unpacker` обычно используется установленный `EFDUnpacker.exe`.

## 2. Headless-режим

Для распаковки без GUI используйте команду:

```bash
efd_unpacker unpack /path/to/file.efd -tmplts /path/to/output_dir
```

Поведение:
- успех: сообщение `[OK] ...`, код возврата `0`
- ошибка: сообщение `[ERROR] ...`, код возврата `1`

Особенности:
- создаёт выходную директорию при необходимости;
- поддерживает абсолютные, относительные и `~` пути;
- использует локализованные сообщения приложения.

Команда `unpack <file>` без `-tmplts <output_dir>` не считается headless-режимом.

## PATH

| Платформа | Вариант поставки | PATH |
|-----------|------------------|------|
| macOS | `DMG` | автоматически после первого запуска установленного `.app` |
| Windows | `setup.exe` | автоматически через установщик |
| Linux | `DEB` | автоматически |
| Linux | `AppImage` | автоматически после первого запуска |

### macOS

После первого запуска уже установленного `EFDUnpacker.app` приложение создаёт пользовательский launcher `efd_unpacker` в управляемом каталоге и добавляет его в `PATH` только пока существуют и launcher, и само приложение.

Важно:
- если приложение запущено прямо с примонтированного `.dmg`, PATH не настраивается;
- после установки в `Applications` просто запустите приложение ещё раз;
- после удаления приложения новая shell-сессия больше не будет добавлять `efd_unpacker` в `PATH`;
- если старая shell-сессия ещё помнит команду, launcher при первом вызове сам удалится;
- для уже открытого терминала может понадобиться новая сессия.

### Linux AppImage

После первого запуска `AppImage` приложение создаёт пользовательский launcher `efd_unpacker` в управляемом каталоге и добавляет его в `PATH` только пока существуют и launcher, и сам `AppImage`.

Если файл `AppImage` был перемещён, просто запустите его ещё раз, чтобы launcher обновился. Если `AppImage` удалён, новая shell-сессия больше не будет добавлять `efd_unpacker` в `PATH`.

### Windows setup.exe

Windows setup добавляет приложение в системный `PATH`, поэтому после установки бинарь можно вызывать из консоли без перехода в каталог установки.

## Совместимость

CLI использует единый прикладной слой и работает одинаково по смыслу на Windows, Linux и macOS.
