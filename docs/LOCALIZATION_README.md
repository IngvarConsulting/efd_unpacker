# Локализация приложения

Документ описывает локализацию самого приложения. Он не описывает язык платформенных установщиков.

## Текущее поведение

- базовый язык приложения: английский;
- если система определена как русская, используется русский;
- если перевода нет, показывается исходная английская строка.

Выбор языка сосредоточен в [src/efd_unpacker/runtime.py](../src/efd_unpacker/runtime.py) через `detect_system_language()`.

## Как устроено

- приложение не использует `QTranslator` и `.qm` для своих строк;
- строки приложения переводятся через `Translator`;
- сообщения CLI и GUI формируются в прикладном слое, а не в доменных сервисах;
- `resource_path()` позволяет искать переводы и ресурсы одинаково из исходников и из PyInstaller bundle.

## Файлы переводов

Сейчас в репозитории есть только:
- `translations/ru.ts`

Отдельный `en.ts` не нужен: английский остаётся исходным текстом по умолчанию.

## Использование в коде

```python
from efd_unpacker.localization.translator import Translator

translator = Translator(lang="ru")
translator.translate("MainWindow", "EFD Unpacker")
```

## Добавление нового языка

1. Создайте `translations/xx.ts`.
2. Обновите шаблон:

```bash
lupdate src/efd_unpacker -ts translations/xx.ts
```

3. Заполните переводы через Qt Linguist.
4. При необходимости расширьте логику автоматического выбора языка.

## Добавление новых строк

1. Добавьте новую исходную строку в код.
2. Обновите `.ts`:

```bash
lupdate src/efd_unpacker -ts translations/ru.ts
```

3. Переведите новые сообщения.

## Packaging

- PyInstaller включает `.ts` и ресурсы в bundle;
- `Translator` и GUI не зависят от текущей рабочей директории;
- если переводы не найдены, приложение остаётся на английском.
