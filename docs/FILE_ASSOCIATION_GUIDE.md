# Файловые ассоциации и интеграция с ОС

Документ описывает, как приложение получает `.efd` файл от операционной системы и открывает его в GUI.

## Что поддерживается

- двойной клик по `.efd`
- открытие файла через системную команду `Open with`
- передача пути к `.efd` в аргументах CLI
- drag and drop в окно приложения
- схема `efd://...`

## Как это работает по платформам

### macOS

- ассоциации и тип файла описаны в bundle metadata;
- приложение получает `FileOpen` события и подставляет файл в форму;
- для обычного пользователя основной путь установки — `DMG`.

### Windows

- setup.exe регистрирует ассоциации через встроенный MSI-пакет WiX;
- двойной клик по `.efd` должен запускать `EFDUnpacker.exe` с путём к файлу.

### Linux

- `DEB` устанавливает `.desktop` файл и MIME-описание;
- `AppImage` может требовать ручной desktop integration в зависимости от окружения;
- desktop entry использует команду `efd_unpacker %f`.

## Ручная проверка

### Аргументы командной строки

```bash
efd_unpacker /path/to/file.efd
efd_unpacker efd:///path/to/file.efd
```

На Windows аналогичная проверка обычно делается через `EFDUnpacker.exe`.

### Linux MIME

При ручной настройке desktop integration можно использовать:

```bash
xdg-mime default efd_unpacker.desktop application/x-efd
```

## Типы и идентификаторы

- расширение: `.efd`
- MIME: `application/x-efd`
- URL scheme: `efd://`
